Links and Images

Links:

Using the a (anchor) tags, along with 'href="a url to link here"'(Fun Fact: href stands for hypertext reference). Let's try a link to The Odin Project now.

There are two different types of URLs to link. The first, Absolute Links, link to pages on other websites. They must contain the protocol like in the link used above(example: https) The second, Relative Links, link to pages within our website. They do not include a protocol, they include the file path to the other page relative to the one created. We can also use the "anchor" tags for more than URLs. They can also linke to videos, pdfs, images, etc.

Commit Messages

Commit messages are notes only seen in the code. They are beneficial when working alone on a project or with a group. They will only been seen in code and are a good reminder of where you left off, when you change something or for any reason that a note would be helpful for your future self or team members working on the same project to communicate changes and/or plans.

Think of commit messages like a "snapshot" of your code at the moment the commit was made.

It is considers best practice to commit every time you have a meaningful change in the code. This creates a timeline of progress.

Some examples of when to make a commit:

  • Fixing a typo
  • Fixing a bug
  • When working on making a function work the way you want it to
  • When you get that function to work!
  • To give context to a change to fellow developers, or to your future self

A well formed dedication to commit messages will ensure a site or project will be easier to maintain for extended periods of time.

The Seven Rules of a great Git commit message:

  1. Separate subject from body with a blank like
  2. Limit the subject like to 50 characters
  3. Capitalize the subject line
  4. Do not end the subject line with a period
  5. Use the imperative mood in the subject line
  6. Wrap the body at 72 characters
  7. Use the body to explain what and why vs. how

Lastly, the commit message will be written in the command line of shell. Ex) git commit -m "Add my first commit"

A properly formed Git commit subject line should always be able to complete the following sentence:

  • If applied, this commit will your subject line here
    • If applied, this commit will Add my first commit
