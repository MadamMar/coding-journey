How to Run JavaScript Code

The simplest way to run JavaScript is within the browser, we'll simply create an HTML file with the JavaScript code inside of it. Just add [style] tags within the head of the code, within write [console.log("Hello, World!")]
Anything placed within the ( ) will print to the developer console in the browser.

To include JavaScript within an external script, it is similar to linking external CSS (as I have been doing thus far). Add [script src="javascript.js"] [/script] . The .js extension works like .css for stylesheets

Variables

A variable is a "named storage" that is used to store all kinds of data/information. To create a variable in JavaScript, use the keyword let.
An example of how let works:
1 let message; (creats/declares a variable with the name "message")
2 message = "Hello"; (store the string "Hello" in the variable named "message". It is the data/info)
3 alert(message); (shows the variable content)
(Line 1 and 2 can be combined as let message = 'Hello';)

Declaring multiple variables in one line would look like this:
1 let user = 'John', age = 25, message = 'Hello';

Though not recommended, for better readability they should be given their own lines like this:
1 let user ='John';
2 let age = 25;
3 let message = 'Hello';
Or more simply:
1 let user = 'John';
2 age = 25;
3 message = 'Hello';

An easy way to think about variables is a way of organization. In the previous exmaple of message, think of a box labelled "message" (the variable) and within the box we put "Hello" (the value).
A variable should only be declared once (using let) and from then on, it should only be used without being declared.

Variable Naming

2 Rules of naming:
The name must contain only letters, digits or the $ and _ symbols.
The first character must not be a digit

When naming a Variable with more than one word, capitalize the beginning of each word (except for the first) like so: thisWordyVariableName
Variables are cast sensitive, you can also use any language/alphabet/hieroglyphs/cyrillic letters.
Some words are reserved and cannot be used as variable names, such as 'let', 'class', 'return', 'function'.

Constants
To make a variable unchangable, declare it a constant using const instead of let.
These variables cannot be reassigned. This communicates clearly that the value will never change (example, a birthday is always the same date)
Constants (which are generally done in uppercase to stand out for a decided value) are useful to simplify a value that is difficult to remember, like with colors. Example:
const COLOR_ORANGE = "#FF7F00";
COLOR_ORANGE is much easier to remember than the hexidecimal format and harder to misstype, especially if you'll be cycling through many different colors.

When values are constant but are calculated in run-time, but don't change after the initial assignment, we do not use all uppercase. Example:
const pageLoadTime =
Simply put, capital named constants are only used as aliases for "hard-coded" values.

Naming Summed up

  • Names should be clean and clear for what is stored inside. It should be easy for another coder to see and understand easily.
  • They should be easily readable like userName, shoppingCart
  • Avoid abbreviations
  • Create new variables for new things as opposed to always reusing the old ones. This will help in situations where debugging is required and is just more organized.
