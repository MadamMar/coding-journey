Basic Git + Git Commit commands and what they do:

  • $git clone SSH key: will pull your Repository from GitHub into your terminal
  • $git status: will show the status of changes made to the directory
  • $git add filename.filetype: will add a desired text to the directory/"staging area"
  • $git commit -m "Your commit message here": will commit changes from the "staging area"
  • $git log: will show a log of changes with many details
  • $git push [origin main]: will upload your work to GitHub

After a file has been altered or created, then saved, the file will appear red when status is checked, ex) modified: filename.filetype

To save the changes, use the '$git add' command to queue it into the staging area, the file will then appear green when status is checked. ex) modified: filename.filetype

At this point, use the '$git commit -m' command to commit the changes made and add a descriptive message about what was changed. When status is checked, you'll receive a message stating nothing to commit, working tree clean

Lastly, use the '$git push' command to upload the changes to GitHub

CSS time, let's get stylish ...

Selectors: are HTML elements that follow the same rules as CSS, they are what is being selected for each rule.

The Most Common Selectors:

  • Universal Selectors
    • Universally selects an element of any type and add the selector to it, example:

      • * {

      color: purple;

      }

      All fonts on the page would appear purple.

  • Type Selectors (or Element Selectors)
    • Selects all elements of the same type, example:

      • div {

      color: white;

      }

      All 'div' elements will appear white

  • Class Selectors
      • Selects all elements with the given class, it is an attribute that can be added to HTML, example:

      .example {

      color: red;

      }

      All specified code will appear in red if 'class="example"' is in the element (Note. multiple classes can be added to one Attribute)

  • ID Selectors
    • Selects all elements with the given id, similar to Class Selectors, it is an attribute added to HTML. Unlike Class, an element can only have one ID (Note. it should be used sparingly), example:

      • #example {

      background-color: red;

      }

      All specified code will appear with a black background if 'id="example' is in the element, example:

  • Grouping Selectors
    • Allows multiple selectors to be used together if they share the same declarations using a comma, example:

      • .example-one,

      .example-two {

      color: white;

      }

      It cuts down on repetition, each example can still have more declarations in another selector, but this way repetition only needs to be written once

  • Chaining Selectors
    • Allows selectors to be chained as a list, example:

      • .example-one.example-two { OR .example-one#example-two {

      color: red;

      {

      These selectors must share the same element type, like 'div' or 'p'

CSS Properties to start with:

  • color: sets an element's text color, ex) #1100ff; rgb(100, 0, 127); hsl(15, 82%, 56%);
  • background-color: sets an element's background color, similar to text
  • font-family: sets an element's font, ex) "Times New Roman", sans-serif; (multiple should be listed)
  • font-size: sets an element's font size, ex) 22px;
  • font-weight: sets an element's boldness of text, ex) bold; 700;
  • text-align: sets an element's horizontal alignment, ex) center;
