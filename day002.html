Basic Git + Git Commit commands and what they do:

  • $git clone SSH key: will pull your Repository from GitHub into your terminal
  • $git status: will show the status of changes made to the directory
  • $git add filename.filetype: will add a desired text to the directory/"staging area"
  • $git commit -m "Your commit message here": will commit changes from the "staging area"
  • $git log: will show a log of changes with many details
  • $git push [origin main]: will upload your work to GitHub

After a file has been altered or created, then saved, the file will appear red when status is checked, ex) modified: filename.filetype

To save the changes, use the '$git add' command to queue it into the staging area, the file will then appear green when status is checked. ex) modified: filename.filetype

At this point, use the '$git commit -m' command to commit the changes made and add a descriptive message about what was changed. When status is checked, you'll receive a message stating nothing to commit, working tree clean

Lastly, use the '$git push' command to upload the changes to GitHub
